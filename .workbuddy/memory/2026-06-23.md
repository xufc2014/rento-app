# 2026-06-23 工作日志

## Android APK 导出/导入 bug 修复

三个 bug 都是打包成 Android APK 后出现的：

### Bug 1 — 导出 Excel 点击无反应
- **文件**: `utils/export.js`
- **根因 1**: `saveAsCSV_App` 用了 `plus.io.PUBLIC_DOCUMENTS`（不存在的常量），Android 上静默失败
- **根因 2**: `writer.write(content)` 先于 `writer.onwrite` 回调赋值，异步竞态导致回调永不触发
- **修复**: 改用 `plus.io.PUBLIC_DOWNLOADS`；先赋值回调再 write；失败时自动降级到 `plus.io.PRIVATE_DOC`；新增 `saveToPrivate()` 兜底函数；弹 Modal 展示完整文件路径

### Bug 2 — 导出配置提示保存成功但找不到文件
- **文件**: `pages/settings/index.vue`
- **根因**: Modal 只显示文件名不显示路径，用户不知道去哪找
- **修复**: 用 `plus.io.convertLocalFileSystemURL()` 获取真实路径并展示；写入成功后弹 "是否分享" 选择框 → 确认则调用 `plus.share.sendWithSystem({type:'file'})` 系统分享面板；写入失败自动降级到私有目录

### Bug 3 — 导入配置点击无反应
- **文件**: `pages/settings/index.vue`
- **根因**: `document.createElement('input')` + `input.click()` + 立即 `removeChild(input)` — Android WebView 中文件选择弹窗是异步的，用户选择文件时 input 节点已销毁，change 事件丢失
- **修复**: 
  - `removeChild` 移到 `onchange` 回调内部执行（选完文件后再移除）
  - `click()` 包裹在 `setTimeout(100ms)` 中确保 DOM 挂载完成
  - H5 版本同步修改保持一致

## 模拟器实测修复 — 导入仍然报错
- **现象**: 导出CSV和JSON正常，但导入报 `TypeError: Cannot read property 'createElement' of undefined`
- **根因**: uni-app APP模式下 `<script>` 在独立JS引擎(JSCore/V8)中执行，不在WebView渲染线程中，根本不存在 `document` 对象。之前 Bug 3 的修复方案 (`document.createElement`) 对 APP 端无效
- **正确方案**: APP端使用 `uni.chooseFile()` + `plus.io.resolveLocalFileSystemURL()` + `plus.io.FileReader()`
- H5 端保持 `document.createElement` 不变（H5运行时 JS 在浏览器主线程，document 可用）

## uni.chooseFile 不可用 → 三级降级方案
- HBuilderX 4.45 模拟器上 `uni.chooseFile` 不可用，需降级到原生 Android Intent
- **三级降级**: uni.chooseFile() → plus.android Intent (ACTION_OPEN_DOCUMENT) → 手动粘贴提示
- Intent 文件选择器初始用 `application/json` 作 MIME 类型，导致部分文件管理器过滤掉 data 目录
- **修复**: Intent MIME 类型改为 `*/*` 显示所有文件，同时保留 JSON 相关类型的 EXTRA_MIME_TYPES
- **语法错误修复**: showPasteImportFallback 下多余的 `}` 导致编译失败
