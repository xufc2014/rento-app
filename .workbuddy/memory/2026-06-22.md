## Bug 修复（2026-06-22 第二轮）
- **APP-PLUS 导入**：`plus.io.resolveLocalFileSystemURL` + `plus.io.FileReader` 在 Android 10+ 无法读取 `content://` URI → 改用 `uni.getFileSystemManager().readFile()`，兼容两种 URI 格式
- **APP-PLUS 导出分享**：原来 `plus.share.sendWithSystem({type:'text'})` 只分享文字不发文件 → Android 改为 `type:'file'` 直接分享文件；iOS 弹窗提示 iTunes 文件共享路径

## Bug 修复（2026-06-22 第三轮 — H5 导入报错）
- **H5 导入 `importInput.value.click is not a function`**：uni-app H5 模式中 `ref` 拿到的是 Vue 包装对象而非原生 DOM 节点 → 改用 `document.getElementById`。
- **H5 导入仍不弹文件选择框**：模板内 `<!-- #ifdef H5 -->` 条件编译在 Vite dev 模式下不可靠，且 uni-app 可能不渲染原生 `<input>` → 最终方案：`triggerImport()` 内用 `document.createElement('input')` 动态创建 file input，每次点击都是全新元素，点击后自动移除。彻底绕开模板渲染问题。

## Bug 修复（2026-06-22 第四轮 — 水电单价输入框不显示）
- **问题**：设置页水电单价配置区域，`¥` 前缀可见但输入框不显示
- **根因**：uni-app H5 模式中 `<input class="form-input">` 渲染为 `<uni-input class="form-input"><div class="uni-input-wrapper"><input .../></div></uni-input>`，`.form-input` class 挂在 `<uni-input>` 外层，但 `flex: 1; height: 100%` 没有穿透到内部 native `<input>`，导致输入框宽度塌陷为 0
- **修复**：`.input-wrap-sm` 内新增 `:deep(input)` 和 `:deep(.uni-input-input)` 选择器直接给 native input 设置 `flex:1; width:0; min-width:0; height:100%`；同时给 `.input-prefix` 加 `flex-shrink:0` 防止被挤没
- **仍不显示 → 最终改为纵向布局**：横排三列太挤且 uni-app `<input>` 组件在横排 flex 中宽度容易塌陷。改为和"费用配置"区域一致的纵向 .form-group 布局（标签在上，带 ¥ 前缀的 input 在下），彻底复用已验证的 `.input-wrap` 样式

## Bug 修复（2026-06-22 第五轮 — APP 模拟器导出/导入报错）
- **APP 导出报错「无法访问SD卡」**：`plus.share.sendWithSystem({type:'file'})` 在 Android 模拟器上不支持分享文件。改为保存到 `plus.io.PUBLIC_DOWNLOADS` 公开下载目录，弹 Modal 提示文件名和路径，用户自行到文件管理找。
- **APP 导入报错 `uni.chooseFile is not a function`**：uni-app 无此 API（需打包真机才有）。APP 端也改用 `document.createElement('input')` 动态创建 file input（与 H5 一致），`onFileSelected` 移除 `#ifdef H5` 条件编译，复用同一套 FileReader 逻辑。
